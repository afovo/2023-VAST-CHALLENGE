Based on this theory, we set up a model considering different aspects to calculate the anomaly score for companies, persons, organizations, and nodes with unknown types. The formula is:

 

V stands for the vessel, n stands for nodes, dist stands for Euclidean distance function, Oceanfront is. short for Oceanfront Oasis Inc Carriers, and wcc is short for weakly connected components. 4 - Reflection: What was the most difficult aspect of working with this knowledge graph? Did you have the tools and resources you needed to complete the challenge? What additional resources would have helped you? Limit your response to 300 words  

 

In the process of manipulating the knowledge graph, we identified numerous irrational triples, like nodes labeled ¡°movement¡± linked with ¡°location¡± nodes via family relationship edges, and ¡°event¡±-type nodes connected to ¡°company¡±-type nodes via ownership edges. Over one-fifth of node types remained undefined, potentially indicating issues in text-to-knowledge-graph algorithms.

 

Given the unreliability of the knowledge graph¡¯s information and the absence of original text, it became particularly challenging to convert node and edge data into comprehensible semantic information, necessitating significant time and effort to analyze this information while aligning it with the definition of IUU fishing.

 

To better align with the challenge¡¯s analytical requirements, we engineered an flexible visualization system utilizing tools like d3.js, ECharts, and Vue3. Our system delivers interactive, user-friendly pages featuring main views comprising ego-net, unity-net, hierarchy tree, and parallel coordinates plot that facilitate graph exploration via connection relations, component information, tree structures, and neighboring node attribute values.

 

Utilizing open-sourced data analysis algorithms greatly aided our back-end data analysis, with algorithms such as LINE enabling node similarity observation. The LINE algorithm operates on principles of maintaining node similarity and embedding, eventually yielding first- and second-order embedding vectors for each node. Although various anomaly detection algorithms we experimented with were not ultimately incorporated to our solution, they nonetheless served as valuable inspiration in our pursuit of innovative methods to identify exploitable nodes.
